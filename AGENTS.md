## 스택 & 실행

- 개발: `npm run dev` (http://localhost:5173). 빌드: `npm run build` (= `tsc -b && vite build`). 린트: `npm run lint`.
- 데이터: 지금은 `src/data/events.ts` 시드. 추후 외부 소스/API로 교체.
- 시드 "오늘"은 `App.tsx`의 `TODAY` 상수(`2026-06-05`). 실데이터 연동 시 `new Date()`로 교체.

## 코딩 컨벤션
- 주변 코드와 같은 스타일·주석 밀도. 컴포넌트는 작게, 타입은 `src/types.ts`에.
- 데이터와 표현 분리: 이벤트는 데이터, 로직은 컴포넌트.

- 날짜는 `src/lib/date.ts`의 **TZ 안전 문자열 헬퍼**만 사용 — `new Date(...)` 직접 파싱으로 타임존 버그 내지 말 것.

## 절대 규칙 (Non-negotiables, spec §4와 동일)
1. **사실(일정·점수·홈/원정·장소)은 추론하지 말고 출처로 필드마다 검증.** 패턴을 사실처럼 단정 금지. 불확실하면 표시.
2. 신뢰 출처 우선(공식 리그/팀/대회 사이트, ESPN, Wikipedia 등). SEO 스팸·리세일 사이트 금지.
3. 홈/원정·시드·시간(타임존)은 특히 틀리기 쉬움 — 종원님이 스폿체크할 수 있게 불확실성을 드러낼 것.
4. **⏰ 모든 `time` 필드는 "그 경기 당일 뉴욕(America/New_York) 벽시계 시각".** 앱이 "ET"로 고정 표시하므로 경기장 현지 시각을 그대로 넣으면 틀린다. ET는 EDT(여름·UTC−4)·EST(겨울·UTC−5)를 포괄하니 값이 그 날짜의 뉴욕 시각이면 항상 맞다. **⚠️ UTC 오프셋을 −4/−5로 손으로 고정하지 말 것(서머타임 따라 다름).** 환산: ① 출처가 ET 시각을 직접 주면 그대로 사용, ② UTC만 있으면 `new Date("...Z").toLocaleString("en-US",{timeZone:"America/New_York"})`로 변환(서머타임 자동). 원래 현지/한국 시각은 description에 남긴다.
   **🔒 시각 있는 경기는 반드시 `at("UTC", "src")` 두 인자로 넣는다.** `src` = 출처가 표기한 현지 시각을 **그대로 옮겨 적은 것** — `"YYYY-MM-DD HH:mm IANA타임존"` (예 `"2026-07-29 19:00 Asia/Seoul"`, `"2026-07-15 11:00 Europe/Paris"`). `scripts/check-times.mjs`(빌드 맨 앞에서 자동 실행, `npm run check:times`)가 UTC↔src를 기계 검증해 안 맞으면 **빌드 실패 → 커밋 불가**. 즉 UTC 환산을 틀려도 시스템이 잡는다 — src에는 절대 "암산한 결과"를 넣지 말고 출처 화면에 보이는 시각을 넣을 것. src 없는 리터럴 `at()`도 빌드 실패한다.
5. 라이트 테마 고정.
